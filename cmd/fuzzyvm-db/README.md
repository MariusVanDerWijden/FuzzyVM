# fuzzyvm-db

Builds and inspects a content-addressed database of EVM bytecodes. Programs are
generated by fuzzing, minimized, and stored keyed by the SHA-256 of their
bytecode (so the database deduplicates).

## Build

```sh
go build -o fuzzyvm-db ./cmd/fuzzyvm-db
```

`generate` shells out to `go test -fuzz`, so the Go toolchain and the FuzzyVM
source must be available at runtime.

## generate

Coverage-guided fuzzing that fills the database, scaling across CPU cores.

```sh
# one worker per CPU, into ./fuzzyvm-db.pebble, until Ctrl-C
fuzzyvm-db generate

# 10 minutes, 8 workers, custom path
fuzzyvm-db generate --db /data/corpus.pebble --procs 8 --time 10m
```

- `--db` (default `fuzzyvm-db.pebble`): database path, created if missing.
- `--procs`/`-p` (default `0`): parallel workers; `0` = one per CPU.
- `--time` (default `0`): duration (`30s`, `10m`); `0` = until interrupted.

## inspect

Prints statistics about an existing database (opened read-only).

```sh
fuzzyvm-db inspect --db /data/corpus.pebble
```

- `--db` (default `fuzzyvm-db.pebble`): database path.
